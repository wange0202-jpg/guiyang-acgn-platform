# 贵阳二次元Cosponsor交流平台 - 设计规范

## 1. 概念与愿景

这是一个专为贵阳地区二次元爱好者打造的综合性交流平台，融合了漫展信息、COS作品展示、服务交易和二手物品交易四大核心功能。整体风格采用二次元动漫美学，色彩鲜艳活泼但不失专业感，营造出充满活力又不失秩序的社区氛围。

## 2. 设计语言

### 美学方向
- **风格**: 新拟态 + 玻璃态 + 柔和渐变，融入二次元元素
- **参考**: Pixiv、FurAffinity、Bangumi的视觉风格

### 色彩体系
```
Primary:      #6366F1 (靛蓝紫 - 主品牌色)
Secondary:    #EC4899 (樱花粉 - 强调色)
Accent:       #F59E0B (琥珀金 - 高亮)
Background:   #0F172A (深空蓝 - 暗色背景)
Surface:      #1E293B (石板灰 - 卡片背景)
Text Primary: #F8FAFC (雪白 - 主文字)
Text Muted:   #94A3B8 (银灰 - 次要文字)
Success:      #10B981 (翡翠绿)
Warning:      #F59E0B (琥珀金)
Error:        #EF4444 (珊瑚红)
```

### 字体系统
- **标题字体**: "Noto Sans SC", sans-serif (思源黑体)
- **正文字体**: "Inter", "Noto Sans SC", sans-serif
- **装饰字体**: "Ma Shan Zheng", cursive (马善政楷书 - 二次元风)

### 间距系统
- 基础单位: 4px
- 常用间距: 8px, 12px, 16px, 24px, 32px, 48px, 64px
- 卡片圆角: 12px
- 按钮圆角: 8px
- 容器最大宽度: 1280px

### 动效哲学
- 页面切换: 淡入淡出 300ms ease-out
- 卡片悬停: scale(1.02) + shadow提升 200ms
- 按钮点击: scale(0.98) 100ms
- 图片加载: 骨架屏占位 + 渐显 400ms

## 4. 功能与交互

### 4.1 漫展专区 (Anime Convention Zone)
**分类**: 贵阳（单一分类，由站主统一发布）

**功能**:
- 漫展列表展示（卡片形式）
  - 显示：封面图、漫展名称、时间、地点、主办方
  - 状态标签：即将开始 / 正在进行 / 已结束
- 发布漫展（站主专属）
  - 表单字段：漫展名称、开始日期、结束日期、地点、主办方信息、详情描述、图片上传
  - 图片支持多张上传，预览功能

**交互细节**:
- 悬停卡片：图片轻微放大，显示"查看详情"按钮
- 点击卡片：进入漫展详情页
- 筛选：按时间筛选（全部/近期/历史）

### 4.2 COS专区 (COS & Photography Zone)
**分类**: COS、汉服(Lolita)、JK、兽装/皮套

**功能**:
- 瀑布流作品展示
- 发布作品（需登录）
  - 表单字段：标题、作品图片（最多9张）、分类标签、作品描述
- 作品详情页
  - 大图查看、点赞、评论、分享
- 用户主页（展示该用户的所有作品）

**交互细节**:
- 瀑布流无限滚动加载
- 图片懒加载 + 骨架屏
- 双击图片触发点赞

### 4.3 服务专区 (Service Marketplace)
**分类**: 妆娘(化妆)、毛娘(假发造型)、摄影师、道具师

**功能**:
- 服务商列表展示
  - 显示：头像、服务类型、昵称、简介、联系方式
  - 评分和评价数量
- 发布服务（需登录）
  - 表单字段：服务类型、昵称/艺名、联系方式、服务介绍、价格区间、作品展示
- 服务详情页
  - 服务商信息、详细描述、作品图册、预约/联系

**交互细节**:
- 分类Tab切换
- 服务商卡片悬停显示联系方式快捷操作

### 4.4 交易专区 (Trading Post)
**分类**: C服(Cosplay服装)、假发、道具、周边/其他

**功能**:
- 商品列表展示
  - 显示：商品图、价格、新旧程度、卖家信息
  - 筛选：分类、价格区间、新旧程度
- 发布商品（需登录）
  - 表单字段：商品名称、分类、价格、商品描述、实物图片（最多9张）、新旧程度、发货方式
- 商品详情页
  - 大图查看、联系卖家、收藏商品

**交互细节**:
- 价格醒目显示（红色高亮）
- "联系卖家"一键复制联系方式

### 4.5 站主人员管理 (Admin User Management)
**功能**（仅站主可见）:
- 查看所有注册用户列表（实时更新）
- 显示每个用户的帖子数量
- 删除普通用户账号
- 删除账号后自动清除：
  - 该用户发布的所有帖子
  - 该用户的点赞记录
  - 该用户的关注数据

**交互细节**:
- 删除前需确认，显示影响范围
- 站主账号无法被删除
- 用户列表实时监听数据变化

### 4.6 实时数据同步
**所有个人主页（帖子、粉丝、关注、获赞）**:
- 帖子数量实时更新（监听 postsChanged 事件）
- 粉丝数量实时更新（监听 followDataChanged 事件）
- 关注数量实时更新（监听 followDataChanged 事件）
- 获赞数量实时更新（基于帖子数据自动计算）
- 页面间切换时自动刷新最新数据

### 4.7 站主审核通知
**功能**:
- Header 导航栏显示待审核内容数量（红色角标）
- 支持的审核类型：COS作品、服务、商品、评论
- 数量实时更新，发布新内容后自动刷新

**实时更新机制**:
- 发布/审核/删除操作后触发对应事件（cosWorksChanged, servicesChanged, productsChanged, commentsChanged）
- 页面自动刷新待审核数量

## 5. 组件清单

### 导航组件
- **Header**: 固定顶部，毛玻璃效果，Logo + 菜单 + 搜索 + 用户头像
- **MobileNav**: 底部Tab导航，移动端专用
- **Breadcrumb**: 面包屑导航（详情页使用）

### 内容展示
- **ZoneCard**: 专区入口卡片，大图标 + 标题 + 简介
- **ActivityCard**: 漫展/活动卡片，封面 + 标题 + 日期 + 状态标签
- **WorkCard**: 作品卡片（瀑布流），图片 + 标题 + 作者头像
- **ServiceCard**: 服务商卡片，头像 + 昵称 + 服务类型 + 评分
- **ProductCard**: 商品卡片，商品图 + 价格 + 标题 + 新旧程度

### 表单组件
- **PostForm**: 通用发布表单，包含标题、内容、图片上传
- **ImageUploader**: 图片上传组件，支持拖拽、多选、预览、删除
- **CategorySelector**: 分类选择器，支持单选/多选
- **DateRangePicker**: 日期范围选择器（漫展专用）

### 反馈组件
- **Toast**: 轻提示，成功/失败/警告
- **Modal**: 模态框，确认操作/表单填写
- **Skeleton**: 骨架屏，加载占位

### 状态
- **Empty**: 空状态，无数据时显示
- **Loading**: 加载中，骨架屏或spinner
- **Error**: 错误状态，提供重试按钮

## 6. 技术方案

### 前端架构
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **样式**: Tailwind CSS + shadcn/ui
- **图标**: Lucide React
- **路由**: React Router DOM
- **状态管理**: React Context + LocalStorage (简化版)
- **图片处理**: 统一上传至图床，本地预览使用Blob URL

### 数据存储
由于是初期版本，采用本地LocalStorage存储模拟数据：
- 用户数据
- 漫展数据（预设 + 站主添加）
- 作品数据
- 服务商数据
- 商品数据

### 页面路由
```
/                     首页（四大专区入口）
/convention           漫展专区
/convention/post      发布漫展（站主）
/convention/:id       漫展详情
/cos                  COS专区
/cos/post             发布COS作品
/cos/:id              作品详情
/cos/user/:userId     用户主页
/service              服务专区
/service/post         发布服务
/service/:id          服务详情
/trading              交易专区
/trading/post         发布商品
/trading/:id          商品详情
```

### 图片存储策略
- 开发环境：使用浏览器Blob + FileReader本地预览
- 生产部署：需接入云存储（预留接口）
